Bug reporting for brickOS

General interactive support is handled via the news group at LUGNET: lugnet.robotics.rcx.legos.  If you wish to tell the brickOS development team of issues with formal or test releases then please follow the procedures on this page.

Reviewing what's been reported

Before filing a bug report, or enhancement request, you should review the existing reports to see if your issue is already covered.  Since we have recently changed names, there are two sets of locations to search: the original legOS site and the new brickOS site.  Here are the links directly to the types of reports filed at both locations:

Review current observations at LUGNET.
Review posts at legOS at LUGNET.
 
Review bugs with fixes (Patches)
Patches at legOS, at brickOS
 
Review bugs without fixes (Bugs)
Patches at legOS, at brickOS
 
Review Enhancement requests
Enhancements at legOS (skip this there weren't any, it's here to be complete.  ;-), at brickOS
 

Reporting what you've found

After reviewing (above section) the open bugs/patches/enhancement requests and finding that you want to report something new, you should choose from one of the following locations to post based on what you've found and how much work you've already put into finding a solution:

Observations without fixes
Simply post these to legOS at LUGNET as new threads.
 
You've attempted a fix for a bug and here's the patch
View and post Patches at brickOS project site Select "submit new" (when you get there) to submit a new patch
 
You found a bug not yet reported
View and post Bugs at brickOS project site Select "submit new" (when you get there) to submit a new bug report
 
You would like to file a request for an enhancement
View and post Enhancement requests at brickOS project site Select "submit new" (when you get there) to submit a new enhancement request
